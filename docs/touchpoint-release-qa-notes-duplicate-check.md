# Touchpoint Brand Review Notes

This file replaces a temporary duplicate-check placeholder with useful review notes.

## Client source direction

The supplied Touchpoint materials position the product as:

- Touchpoint by New Wave Synergy™;
- a Post-Completion, Handover and Asset Transition Platform;
- the operating system for everything after Practical Completion;
- a people, process and technology platform;
- focused on AU + NZ;
- built for developers, builders, asset owners, facilities managers, purchasers and end users.

## Brand direction

Touchpoint should feel like a product under the New Wave Synergy umbrella, not a disconnected startup site.

Keep:

- parent-brand attribution;
- NWS green as the signature colour;
- clear post-completion / handover / asset-transition positioning;
- direct founder/contact credibility;
- professional built-environment language.

Avoid:

- generic SaaS copy;
- abstract AI-style visuals;
- shortening the approved client copy into vague slogans;
- removing New Wave Synergy cues;
- changing the brand relationship without approval.

## Website review priorities

- Hero should better match the capability statement: “Construction Doesn’t End at Practical Completion.”
- Hero image should feel more substantial on desktop while still scaling safely on mobile.
- Logo/wordmark should be reviewed against the PDF/capability-statement lockup.
- Footer should keep a strong NWS attribution and feel consistent with the wider NWS ecosystem.
- JSON-LD should be server-rendered for stronger SEO/no-JS resilience.
- Mobile navigation should have a no-JS fallback.
